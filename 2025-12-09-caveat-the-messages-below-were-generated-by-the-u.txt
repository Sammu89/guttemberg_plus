   static layout values, not user-customizable properties, so they should be
   defined once.

  Best solution would be to:

  1. Define padding in the schema as defaults (if it should be customizable)
  2. OR define them as constants in a shared file (if they're fixed layout
  values)

  Currently, 1rem 1.5rem padding for .accordion-title and .accordion-content
   is just hardcoded in three places:
  - style.scss (line 53 for title, line 137 for content-inner)
  - edit.js (line 287 for title, line 407 for content)
  - editor.scss (now empty, but we tried to add it)

  Should padding be:

  Option A: Schema-defined attribute (users can customize per block)
  - Add titlePadding and contentPadding to accordion.json
  - Generate from schema
  - Both edit.js and save.js use the same attribute value

  Option B: Shared constant (fixed layout value)
  - Create src/constants.js with ACCORDION_PADDING = '1rem 1.5rem'
  - Import and use everywhere
  - Prevents repetition, single source of truth

  Which approach makes sense for your use case?
